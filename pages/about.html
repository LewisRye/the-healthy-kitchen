About

This website promotes healthy food choices by simulating an interactive 3D kitchen. It showcases detailed models of nutritious foods with educational insights and visual appeal.

Accessibility

Accessibility has been considered for this website. An example of this is the responsive design, that UI when you resize the window or use the page on mobile. The friendly design means this website can run on nearly any device. For disability issues, I have made buttons very large with an appropriate colour scheme for people with vision issues.

Design

To ensure maintainability, scalability, and clear separation of concerns, this application follows the Model-View-Controller (MVC) design pattern.

Model - Backend API

ExpressJS was chosen for its simplicity and minimalistic design which is perfect for creating small APIs quickly. The point of this API is to serve nutrition information about the different foods. This backend API communicates with an SQLite database to store the data, and is accessed in the frontend via HTTP GET requests.

View - Bootstrap

Bootstrap 5 was selected for its robust grid system. The consistent dark theme enhances readability and visual appeal, while Bootstrap's grid system ensure that the application looks and functions well on all devices, including on mobile.

Controller - Three.JS

Three.JS was selected as the core rendering engine for the 3D graphics in the application. The decision to use Three stems from its performance and ease of use to create good 3D scenes with animations.

3D Models

Fridge

The fridge opens and closes interactively, revealing foods inside. Animations and sound effects enhance realism. There are also glass shelves inside the fridge that are transparent, making it a more detailed mesh with realistic properties.

Apple

The apple model can be zoomed and spun using mouse interaction, giving a tactile experience.

Milk

A carton of milk that contains drawn textures. There is a separate lid that makes the carton look more realistic.

Banana

A textured 3D model of a banana. The texture of the banana has been drawn in Blender to include slight brown spots, enhancing realism involving the ripeness of the banana. The banana can also be cut in half by the user, increasing interactivity.

3D App Implementation

Design Quality and Usability

The site makes use of Bootstrap's row and column grid layout to ensure responsiveness across devices, such as mobile, and consistent UI appearance/behaviour. Bootstrap and CSS are also used to style the buttons and navigation bars. The site is made more usable by the tooltips that are shown when you hover over something that is clickable.

Animations

Animations can be triggered through clicking objects. For example, the fridge will open when clicked, and the banana will cut in half. There are also buttons such as 'Toggle Wireframe' that the user can use to see the wireframe version of the models.

Textures

Another interesting feature is the 'Redecorate' button, this switches the textures being used on the walls, floor and door and simulates a total redecoration of the kitchen.

Audio and Sound Effects

All sound effects are copyright-free and sourced from Mixkit. The sound effects I have chosen to include are played when opening and closing the fridge, as well as the light switch being toggled, you can try out these sounds by clicking the fridge or the light switch.

Lighting

Lighting is also controlled by buttons. There is a light switch found on the wall that can turn the scene from day into night, and this turns off the sunlight, and turns on a spotlight in the ceiling and puts stars in the sky outside, simulating night time.

Interaction with 3D Models

JavaScript is used to manipulate the 3D models, this is through the camera panning when the user moves the mouse, and when animations are triggered upon clicking an object.

Interesting Features

Drawing Model Textures

The models I have created in Blender are of sufficient complexity, with correct roughness and metallic properties. I have created the textures myself using the 'Texture Paint' tool.

GitHub Archive

This project is version-controlled using Git, with full commit history available on the GitHub repository.

jQuery

jQuery handles dynamic interactivity such as showing and hiding buttons during interactions, an example of this is the 'Close Fridge' button that only shows when the fridge is open.

Testing

Interaction

Interactive elements were tested thoroughly, and are working correctly. Examples of interactivity include the fridge, which opens and closes, the light switch, which can be toggled by the user, and the cupboards which take the user to a chopping board scene where they can cut open a banana.

Upon testing the opening of the fridge, I found that you could click other elements that should not be clicked, such as the light switch. This was fixed by disabling all other buttons when the fridge was open.
